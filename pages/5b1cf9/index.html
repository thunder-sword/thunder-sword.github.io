    # 描述

    通过Pydiction，您可以在Vim中制表符完整的Python代码，例如关键字，内置函数，标准库和第三方模块。

    它不需要安装任何依赖项。它仅包含三个主要文件：

    python_pydiction.vim  -- Vim plugin that autocompletes Python code.
complete-dict         -- Dictionary file of Python keywords, modules, etc.
pydiction.py          -- Python script to add more words to complete-dict.
    1
    2
    3

    捆绑的字典附带了您日常Python编程中可能需要的大多数功能，并且随附的Python脚本使您可以轻松地将新模块附加到字典中。因此，您不必等待我添加它们。而且，您可以教Pydiction很快完成项目自己的API。已经支持的一些第三方库包括： Django Flask Requests Twisted Numpy Psycopg2 PyGreSQL SQLite3 MySQLdb OpenGL Pygame wxPython PyGTK PyQT4 OpenID Scrapy Celery和更多。

    由于Pydiction仅使用平面词典文件，因此它非常灵活，因为您可以执行诸如重新排序希望其在弹出式完成菜单中出现的方式之类的事情。默认情况下，它将按字母顺序排列，但是如果您想else排在前面elif，你可以(调换一下)。

    与其他形式的python代码完成方式相比，拼写常常被误解。Pydiction不必进行任何源代码分析。它仅使用术语词典。这是单独使用时的优点和缺点。这是一种优势，因为它允许插件保持稳定。当完成第三方库和框架以及基本关键字时，Pydiction确实会发光，但是对于字典完成不适合的事情则不适用。为此，您还需要全能。有关Tips如何充分利用世界的信息，请参见本部分。

    # 关于python_pydiction.vim

    Usage (vim)如果您只是想知道如何在Vim中使用Pydiction，请参见本节。本节将详细介绍此插件在后台的功能。

    Pydiction 1.0及更高版本使用名为python_pydiction.vim的文件（位于after / ftplugin /目录中），该文件是ftplugin，仅在您编辑python文件（例如，使用.py扩展名或您手动输入:set filetype=python）。

    这是python_pydiction.vim读取和pydiction.py写入的字典文件。没有此文件，Pydiction将不知道它可以使用Tab键完成哪些Python关键字，内置程序和模块。

    # 拼写与其他形式的完成

    • Pydiction不需要安装任何依赖项，甚至不需要将Python支持编译到您的Vim版本中。基于字典的补全非常稳定，因为它是Vim固有的，并且Pydiction插件只有大约50行代码，并且不太可能出现错误-不计算与其他插件的潜在冲突，但是这些通常很容易修复。
    • 由于Pydiction使用字典文件，因此无需先导入模块即可完成它，甚至不必在计算机上安装该模块。这使得完成非常快，因为它不需要执行任何类型推断。
    • 而且由于字典文件只是静态文本文件，因此您可以重新安排希望东西出现在弹出菜单中的方式。默认情况下，事情都按字母顺序排序，但如果欧要else来之前elif，你可以自定义。
    • 词典文件只有一个全局实例，这意味着pydiction不会尝试在项目文件夹中动态创建词典。因此，您无需添加到.gitignores或其他任何项目的项目文件夹。而且，让Pydiction与virtualenv其他操作系统一起使用无需执行任何特殊操作。
    • 它可以完成Python关键字，内置函数以及Python模块名称及其属性和方法。它可以同时填写完全限定名称和非完全限定名称。例如：string.upper(，upper(，''.upper(，等等。
    • Pydiction仅使用“ Tab”键完成操作，使用特殊的字典文件完成操作，并且仅在编辑Python文件时尝试完成操作。这样的好处是只需要一个按键即可完成操作，并且不会污染您可能用于其他类型的完成操作的所有完成菜单，例如Vim的常规omni-completion或您可能正在运行的其他完成脚本。
    • 由于Pydiction使用可能的完成项目的字典文件，因此与其他方法相比，它可以更准确地完成第三方模块。您完全可以控制它可以完成和不能完成的事情。如果无法完成任何操作，则可以使用pydiction.py将新模块的内容添加到字典中，或者可以使用文本编辑器手动添加它们。换句话说，您可以教Pydiction了解它可以完成哪些新功能，就像在任何基于片段的系统中一样—除了片段是用于自动填充单词的其余部分，而不是像SnipMate一样粘贴整个模板。
    • 词典只是一个文本文件，可在所有平台上移植。例如，如果您是金字塔用户，则可以将所有与金字塔相关的模块添加到字典file.py中。最新的默认complete-dict已包含所有标准库，Python关键字和许多第三方模块，例如Django，Twisted，Numpy，Flask，Requests，Pygame，wxPython，PyQT4，PyGTK，Urwid，ZSI，LDAP，MySQLdb，Psycopg2 ，PyGreSQL，OpenId，OpenSSL，Celery，Scrapy，lxml，Pygments等。要查看Pydiction知道的python模块的完整列表，请在Vim中打开complete-dict并运行:g/root modules。
    • 如果愿意，您可以将Pydiction与Vim 7的内置Python完整功能（pythoncomplete.vim）以及其他完成形式（例如SnipMate或Python-mode）结合使用（请参阅“提示”部分）。实际上，他们都能组成一支出色的团队。
    • Pydiction知道何时完成属性和可调用方法。如果它是可调用的，则它将自动插入一个开括号。
    • Tab键将正常工作。如果您要编辑Python文件并且首先键入某些Python模块或关键字的一部分，则Pydiction仅尝试使用Tab键完成Python代码。
