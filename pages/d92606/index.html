# 最初的Shellshock漏洞：(CVE-2014-6271)

该漏洞的原始形式（CVE-2014-6271）涉及一个精心编制的环境变量，该变量包含导出的函数定义，后跟任意命令。Bash在导入函数时错误地执行了尾部命令。[33]可以使用以下命令测试该漏洞：

env x='() { :;}; echo vulnerable' bash -c "echo this is a test"
1

在受该漏洞影响的系统中，由于Bash执行“echo vulnerable”命令，上述命令将显示“vulnerable”一词，该命令嵌入到精心编制的名为“x”的环境变量中。[8][34]

Shellshock的原理已经被探讨了很多，简单总结一下就是，当环境变量X被如此赋值：一个函数定义后面紧跟命令A（echo vulnerable），子进程会先执行命令A。而且，命令A在子进程的权限下被执行。

# 测试语句：

$ env x='() { :;}; echo vulnerable' bash -c "echo this is a test"
vulnerable
this is a test
#如果先出现‘vulnerable’字符串则代表有这个漏洞
#在pwnable.kr中测试则用下面这个（因为bash位置不同）：
shellshock@ubuntu:~$ env x='() { :;}; echo vulnerable' /home/shellshock/bash -c "echo this is a test"                                                                           
vulnerable
this is a test

Florian Weimer and Todd Sabin found this bug (CVE-2014-7186),[8][32] which relates to an out-of-bounds memory access error in the Bash parser code.[38]

An example of the vulnerability, which leverages the use of multiple "<<EOF" declarations (nested "here documents"):

Florian Weimer和Todd Sabin发现了这个bug（CVE-2014-7186），[8][32]它与Bash解析器代码中的越界内存访问错误有关。[38] 利用多个“<<EOF”声明（嵌套“here documents”）的漏洞示例：

bash -c 'true <<EOF <<EOF <<EOF <<EOF <<EOF <<EOF <<EOF <<EOF <<EOF <<EOF <<EOF <<EOF <<EOF <<EOF' ||

# CVE-2014-7187[edit]

Also found by Florian Weimer,[8] CVE-2014-7187 is an off-by-one error in the Bash parser code, allowing out-of-bounds memory access.[39]

An example of the vulnerability, which leverages the use of multiple "done" declarations:

(for x in {1..200} ; do echo "for x$x in ; do :"; done; for x in {1..200} ; do echo done ; done) | bash ||
echo "CVE-2014-7187 vulnerable, word_lineno"
